# Fixed Bitwarden and Vaultwarden login failures

Root causes:
1. Official Bitwarden moved prelogin to the identity host (`identity.bitwarden.com/accounts/prelogin`); api.bitwarden.com returned 404
2. Master password hash was incorrect — now derives master key then PBKDF2(masterKey, password, 1) as Bitwarden expects
3. Token request used snake_case device fields; Bitwarden expects camelCase (`deviceType`, etc.)
4. Argon2 memory now converts Bitwarden MiB to hash-wasm KiB

Prelogin now tries identity first, then api (Vaultwarden supports both). Health checks updated accordingly.
