# Fixed empty vault list after successful login

Sync was returning encrypted ciphers, but client-side decrypt dropped every item (name decrypt failed → count stayed `0`).

Root causes in `app/utils/crypto.ts`:
1. Master-key stretch used `HMAC(key, "enc")` instead of Bitwarden’s HKDF-Expand (`HMAC(key, info || 0x01)`)
2. User key kept only the 32-byte AES half and was stretched again — cipher strings need the full 64-byte SymmetricCryptoKey (enc + mac) with no further stretch
3. Manual PKCS#7 unpadding after Web Crypto AES-CBC (which already strips padding)

After deploy: Lock → Unlock once (or sign out/in) so the correct user key is stored.
