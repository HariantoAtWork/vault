# Dual API integration and self-hosted login fix

Integrated both Bitwarden API specs as Nitro adapters for a remote web client (desktop-app model), without requiring `bw serve`.

- Fixed self-hosted login: URL normalisation, TLS error handling, health check endpoint, test connection on login
- Added `/api/vault/*` Vault Management API adapter (sync, status, lock, list, field getters, CRUD, generate)
- Added `/api/org/*` Public Organisation API proxy with client credentials auth
- Added settings and org admin UI, architecture docs, and TypeScript types
- Removed temporary `.context/` reference files
